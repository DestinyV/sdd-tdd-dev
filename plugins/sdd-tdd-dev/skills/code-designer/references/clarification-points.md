# 设计澄清点指南

在 `code-designer` 阶段1中，需要确认的关键澄清点。

## 前端构建和部署 ⭐（前端/全栈场景）

- 构建产物输出目录？是否需要修改构建配置？
- 静态资源如何部署？（CDN/服务器/Nginx）
- CDN缓存策略？（JS/CSS带hash永久缓存，HTML不缓存）
- 是否需要Service Worker？缓存策略？
- 浏览器兼容性要求？Polyfill策略？
- 构建优化要求？（代码分割、Tree Shaking、压缩）
- Lighthouse性能目标？（LCP < 2.5s、INP < 200ms、CLS < 0.1）

## 前端监控和可观测性 ⭐（前端/全栈场景）

- 错误追踪方案？（Sentry/自研/第三方）
- 核心性能指标？（FCP、LCP、TTFB、首屏时间）
- 用户行为埋点？（PV/UV、按钮点击、表单提交、关键业务路径）
- 告警规则？（JS错误率、页面加载时间）
- 兼容性监控？（浏览器版本覆盖率统计）

## 服务上下游依赖 ⭐

- 本功能涉及哪些上游调用方？（谁调用我们？）
- 本功能依赖哪些下游服务？（我们调用谁？）
- 是否有新增/修改/废弃的API端点？
- 是否有接口契约变更（参数、返回值、错误码）？
- 下游服务不可用时的容错策略是什么？（超时、重试、降级）

## 数据库表设计改动 ⭐

- 是否有新增表？表名、字段、类型、约束？
- 是否有现有表的字段变更？（新增/删除/修改字段）
- 是否有索引变更？（新增/删除索引）
- 是否有数据迁移需求？（历史数据如何处理？）
- 数据一致性要求？（强一致性 / 最终一致性）

## 中间件和基础设施 ⭐

- Redis：是否需要新增缓存？缓存策略（TTL、淘汰策略）？
- Nacos：是否需要新增/修改配置项？
- Docker/K8s：是否需要修改部署配置？

## 项目配置 ⭐

- 是否需要新增/修改环境变量？
- 是否需要新增第三方依赖？

## 日志规范 ⭐

- 关键业务操作需要记录哪些日志？
- 错误日志需要包含哪些上下文？
- 是否需要链路追踪（traceId）？

## 上线方案 ⭐

- 上线执行顺序？（数据库迁移→配置→代码部署）
- 灰度策略？回滚方案？

## 监控方案 ⭐

- 新增API需要监控哪些指标？
- 是否需要新增告警规则？
- 是否需要新增业务指标埋点？

## 后端API依赖

- 需要调用哪些后端API？列出所有API端点
- 各API的用途和调用时机是什么？
- API的返回数据结构是什么？需要获取哪些字段？
- 是否有分页、搜索、筛选等查询需求？
- 错误处理：API如何返回错误信息？
- 认证和授权：API是否需要特殊的鉴权？

## 数据模型清晰

- 涉及的核心数据对象有哪些？
- 各对象之间的关系是什么？（一对多、多对多等）
- 是否需要数据转换或适配？

## 集成依赖

- 是否依赖其他模块或功能？
- 有什么先决条件或基础设施需求？
