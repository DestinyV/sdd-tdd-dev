# 跨项目协作计划模板

> code-designer 阶段 2.6 使用此模板生成跨项目协作计划

## 文件路径

```
spec-dev/{requirement_desc_abstract}/collaboration-plan.md
```

---

## 模板内容

```markdown
# 跨项目协作计划 - [需求名称]

**需求 ID**: [requirement_desc_abstract]
**生成日期**: YYYY-MM-DD
**涉及项目**: [project-a, project-b, project-c]
**协作类型**: [上下游服务 | 前后端分离仓库 | 微服务集群 | Monorepo多Package]

---

## 项目依赖图

```
[以 ASCII 图形式展示项目间依赖关系，示例:]

project-a (上游 — 认证服务)
    ↓ REST: /api/v1/auth
project-b (中游 — 订单服务)
    ↓ REST: /api/v1/orders
project-c (下游 — 前端应用)
```

> 如果是 Monorepo，以 Package 名替代项目名；如果是前后端分离仓库，标注为 `project-frontend` / `project-backend`。

## 执行顺序（基于拓扑排序）

```

  出口门控：[接口实现完成并通过契约测试]
    ↓
阶段2（中间层）：[project-b]
  出口门控：[接口实现完成并通过集成测试]
    ↓
阶段3（应用层）：[project-c]

```

## 回滚协调计划

> 回滚按 **依赖逆序** 执行：下游先回滚，上游后回滚。

| 步骤 | 操作 | 项目 | 前置条件 | 回滚语句 |
|------|------|------|---------|---------|
| 1 | 停止消费方部署 | [project-c] | 告警触发 | [停止部署流水线] |
| 2 | 回滚消费方 | [project-c] | 步骤1 完成 | [代码回滚到上一版本] |
| 3 | 回滚中间层 | [project-b] | 步骤2 完成 | [代码回滚 + 数据库回滚] |
| 4 | 回滚基础层 | [project-a] | 步骤3 完成 | [数据库迁移回滚 + 代码回滚] |

## 注意事项

- **兼容性**: 上游项目变更时，需确认下游项目的兼容影响
- **版本锁定**: 跨项目调用方应锁定 API 版本号，避免隐式升级
- **通知机制**: 每个阶段完成后通知下游项目负责人
- **数据一致性**: 跨项目事务需考虑分布式一致性（SAGA/消息队列）
```
