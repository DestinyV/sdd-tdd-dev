# 后端基础设施任务指南

## 后端基础设施任务清单

| 任务类别 | Task 示例 | 依赖关系 |
|---------|----------|---------|
| 数据库迁移 | 创建表结构、修改字段、创建索引、数据迁移 | 代码开发前/并行 |
| 中间件配置 | Redis缓存配置、Nacos配置项、消息队列 | 代码开发前/并行 |
| 服务依赖 | 下游服务接口对接、上游服务联调 | 依赖下游服务 |
| 日志实现 | 业务日志、错误日志、traceId传递 | 与业务逻辑并行 |
| 监控埋点 | API监控、业务指标、告警配置 | 与业务逻辑并行 |
| 部署配置 | Dockerfile更新、K8s配置、环境变量 | 代码开发后 |

## 后端上线执行顺序

```
上线前准备（独立于开发Task）：
  T-DB: 数据库迁移脚本编写和测试
  T-CONF: Nacos/环境变量配置
  T-DEP: 依赖服务确认已就绪

上线执行（严格按顺序，不可并行）：
  T-MIGRATE: 执行数据库迁移
  T-DEPLOY: 灰度部署 → 验证 → 全量部署
  T-MONITOR: 监控验证（错误率、延迟、业务指标）
  T-VERIFY: 上线后功能验证
```

## 后端回滚任务

```
T-ROLLBACK:
  - 回滚脚本准备（数据库向前兼容或回滚SQL）
  - 旧版本镜像确认
  - 回滚验证步骤
```

## 后端任务依赖图

```
Batch 0（准备阶段）：T-DB(迁移), T-CONF(配置) → 并行
  ↓
Batch 1（核心开发）：T1(数据层), T2(服务层), T3(API层) → 并行
  ↓
Batch 2（集成）：T4(上下游联调), T5(日志), T6(监控) → 并行
  ↓
Batch 3（部署）：T7(Docker/K8s配置) → 执行
  ↓
上线阶段（严格按顺序）：T-MIGRATE → T-DEPLOY → T-MONITOR → T-VERIFY
```

## 后端任务类型

- **数据库** ⭐：表结构、迁移脚本、索引
- **中间件配置** ⭐：Redis缓存、Nacos配置、消息队列
- **日志实现** ⭐：业务日志记录、错误日志、链路追踪
- **监控埋点** ⭐：API监控指标、业务埋点、告警规则
- **部署配置** ⭐：Dockerfile、K8s配置、环境变量
- **上线脚本** ⭐：数据库迁移脚本、配置变更脚本
