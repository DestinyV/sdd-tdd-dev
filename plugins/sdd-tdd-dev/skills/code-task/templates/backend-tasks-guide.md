# 后端基础设施任务指南

## ⭐🆕 全栈模式：数据库与接口契约优先

当项目涉及前后端都改动时，**后端任务必须按以下顺序执行**：

```
阶段1：数据库设计确认（design.md 中数据库章节已完成）
  T-DB-DESIGN: 数据库表结构设计
  T-DB-MIGRATE: 迁移脚本编写
    ↓ (用户确认)
阶段2：接口契约定义（design.md 中接口契约章节已完成）
  T-API-DESIGN: API 端点、请求/响应、错误码定义
  T-API-MOCK: Mock 数据生成
    ↓ (前后端双方确认)
阶段3：后端核心开发（基于已确认的数据库和接口契约）
  T-BE1: 数据层实现（Model/Repository/DAO）
  T-BE2: 服务层实现（Service/Business Logic）
  T-BE3: API 层实现（Controller/Handler）
    ↓
阶段4：集成与联调
  T-INT: 前后端联调验证
```

> **⚠️ 关键约束**：不得在没有确认接口契约的情况下开始前端开发。
> 后端开发的 API 实现必须严格遵循接口契约中定义的请求参数、响应体和错误码。

## 后端基础设施任务清单

| 任务类别 | Task 示例 | 依赖关系 | 优先级 |
|---------|----------|---------|--------|
| 🆕 数据库设计 | 表结构设计、字段定义、约束 | design.md 确认后 | **最高（先执行）** |
| 🆕 接口契约 | API定义、请求/响应、错误码、Mock | 数据库设计确认后 | **最高（先执行）** |
| 数据库迁移 | 创建表结构、修改字段、创建索引、数据迁移 | 接口契约确认后 | 高 |
| 中间件配置 | Redis缓存配置、Nacos配置项、消息队列 | 代码开发前/并行 | 中 |
| 服务依赖 | 下游服务接口对接、上游服务联调 | 依赖下游服务 | 中 |
| 日志实现 | 业务日志、错误日志、traceId传递 | 与业务逻辑并行 | 中 |
| 监控埋点 | API监控、业务指标、告警配置 | 与业务逻辑并行 | 中 |
| 部署配置 | Dockerfile更新、K8s配置、环境变量 | 代码开发后 | 低 |

## 后端上线执行顺序

```
上线前准备（独立于开发Task）：
  T-DB: 数据库迁移脚本编写和测试
  T-CONF: Nacos/环境变量配置
  T-DEP: 依赖服务确认已就绪

上线执行（严格按顺序，不可并行）：
  T-MIGRATE: 执行数据库迁移
  T-DEPLOY: 灰度部署 → 验证 → 全量部署
  T-MONITOR: 监控验证（错误率、延迟、业务指标）
  T-VERIFY: 上线后功能验证
```

## 后端回滚任务

```
T-ROLLBACK:
  - 回滚脚本准备（数据库向前兼容或回滚SQL）
  - 旧版本镜像确认
  - 回滚验证步骤
```

## 后端任务依赖图

### 全栈模式（前后端都改动）⭐🆕

```
Batch 0（准备阶段，必须串行）：
  T-DB-DESIGN: 数据库表结构设计 → 用户确认
    ↓
  T-DB-MIGRATE: 迁移脚本编写
    ↓
  T-API-DESIGN: 接口契约定义（API端点、请求/响应、错误码）
  T-API-MOCK: Mock 数据生成
    ↓ (前后端双方确认接口契约)

Batch 1（后端核心开发）：
  T-BE1(数据层/Repository), T-BE2(服务层/Service), T-BE3(API层/Controller) → 并行
    ↓

Batch 2（集成）：
  T-INT(前后端联调), T-LOG(日志), T-MON(监控) → 并行
    ↓

Batch 3（部署）：
  T-DEPLOY(Docker/K8s配置) → 执行
    ↓

上线阶段（严格按顺序）：
  T-MIGRATE → T-DEPLOY → T-MONITOR → T-VERIFY
```

### 仅后端模式

```
Batch 0（准备阶段）：T-DB(迁移), T-CONF(配置) → 并行
  ↓
Batch 1（核心开发）：T1(数据层), T2(服务层), T3(API层) → 并行
  ↓
Batch 2（集成）：T4(上下游联调), T5(日志), T6(监控) → 并行
  ↓
Batch 3（部署）：T7(Docker/K8s配置) → 执行
  ↓
上线阶段（严格按顺序）：T-MIGRATE → T-DEPLOY → T-MONITOR → T-VERIFY
```

## 后端任务类型

- **🆕 数据库设计** ⭐：表结构、字段、约束（全栈模式下最先完成）
- **🆕 接口契约** ⭐：API定义、请求/响应、错误码、Mock数据（全栈模式下第二完成）
- **数据库** ⭐：表结构、迁移脚本、索引
- **中间件配置** ⭐：Redis缓存、Nacos配置、消息队列
- **日志实现** ⭐：业务日志记录、错误日志、链路追踪
- **监控埋点** ⭐：API监控指标、业务埋点、告警规则
- **部署配置** ⭐：Dockerfile、K8s配置、环境变量
- **上线脚本** ⭐：数据库迁移脚本、配置变更脚本
