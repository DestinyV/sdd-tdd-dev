# 前端基础设施任务指南

## 前端基础设施任务清单

| 任务类别 | Task 示例 | 依赖关系 |
|---------|----------|---------|
| 构建优化 | 代码分割配置、压缩优化、Tree Shaking | 代码开发并行 |
| CDN配置 | 静态资源上传CDN、缓存策略设置 | 构建配置完成后 |
| 错误追踪 | Sentry/监控SDK接入 | 与业务逻辑并行 |
| 性能监控 | Core Web Vitals采集、首屏计时 | 与业务逻辑并行 |
| 用户埋点 | PV/UV、按钮点击、业务路径埋点 | UI组件开发后 |
| 兼容性 | Polyfill配置、浏览器测试 | 代码开发后 |
| **浏览器 E2E 测试** ⭐ | Playwright 端到端测试编写 | 组件实现后并行 |
| **视觉回归测试** ⭐ | 关键页面截图对比配置 | 组件实现后并行 |
| **组件 UI 测试** ⭐ | 组件级交互测试编写 | 组件实现后并行 |

## 前端测试任务说明

每个前端 UI 组件 Task 都应关联对应的浏览器测试 Task：

```
T1: 登录组件实现 → 关联 BT-1: 登录 E2E 测试
T2: 表单验证逻辑 → 关联 BT-2: 表单错误展示 E2E
T3: Dashboard 页面 → 关联 BT-3: Dashboard 视觉回归
```

测试 Task 可以和对应组件 Task 在同一批次中由同一子代理完成。

## 前端上线执行顺序

```
上线前准备（独立于开发Task）：
  T-BUILD: 生产构建验证
  T-CDN: CDN域名和缓存配置确认
  T-COMPAT: 浏览器兼容性验证

上线执行（严格按顺序，不可并行）：
  T-UPLOAD: 构建产物上传CDN
  T-DEPLOY: 部署HTML入口文件
  T-CLEAR: 清除CDN缓存（如需要）
  T-VERIFY: 线上功能验证
  T-MONITOR: 监控数据确认（JS错误率、加载时间）
```

## 前端回滚任务

```
T-FE-ROLLBACK:
  - 上一版本构建产物确认
  - HTML入口文件回滚
  - CDN缓存清除
  - 回滚验证
```

## 前端任务依赖图

```
Batch 0（准备阶段）：T-BUILD-CONF(构建配置) → 执行
  ↓
Batch 1（核心开发）：T1(组件层), T2(状态管理), T3(API集成) → 并行
  ↓
Batch 2（增强）：T4(样式优化), T5(错误追踪), T6(性能监控) → 并行
  ↓
Batch 3（部署）：T-CDN(上传CDN) → 执行
  ↓
上线阶段（严格按顺序）：T-UPLOAD → T-DEPLOY → T-CLEAR → T-VERIFY → T-MONITOR
```

## 前端任务类型

- **构建配置** ⭐：Webpack/Vite配置优化、代码分割、压缩
- **CDN部署** ⭐：静态资源上传CDN、缓存策略配置
- **前端监控** ⭐：错误追踪SDK接入、性能指标采集、用户行为埋点
- **浏览器 E2E 测试** ⭐：Playwright 端到端测试，覆盖完整用户流程
- **视觉回归测试** ⭐：关键页面/组件截图对比
- **组件 UI 测试** ⭐：组件级别渲染和交互测试
