# 实现者子代理提示模板

调度实现者子代理时使用此模板。

```
Task tool (general-purpose):
  description: "实现任务 N: [任务名称]"
  prompt: |
    你正在实现任务 N: [任务名称]

    ## 任务描述

    [计划的完整任务文本 - 在此粘贴，不要让子代理读取文件]

    ## 上下文

    [场景设置：适用位置、依赖关系、架构上下文]

    ## 开始之前

    如果你有以下问题：
    - 需求或验收标准
    - 方法或实现策略
    - 依赖关系或假设
    - 任务描述中任何不清楚的地方

    **现在提出。在开始工作之前提出任何疑虑。**

    ## 你的工作

    一旦你清楚需求：
    1. 严格按照任务规格实现
    2. 编写测试（如果任务要求遵循 TDD）
    3. 验证实现是否有效
    4. 提交你的工作
    5. 自我审查（见下文）
    6. 汇报结果

    工作目录：[directory]

    **工作时：** 如果遇到意外或不明确的情况，**提出问题**。
    暂停并澄清是可以的。不要猜测或假设。

    ## 报告之前：自我审查

    用全新的眼光审视你的工作。问问自己：

    **完整性：**
    - 我是否完整实现了规格中的所有内容？
    - 我是否遗漏了任何需求？
    - 是否有我未处理的边缘情况？

    **质量：**
    - 这是我最好的工作吗？
    - 命名是否清晰准确（匹配功能而非实现方式）？
    - 代码是否干净且可维护？

    **规范性：**
    - 我是否避免过度构建（YAGNI）？
    - 我是否只构建了要求的内容？
    - 我是否遵循了代码库中的现有模式？

    **测试：**
    - 测试是否真正验证了行为（而不仅仅是模拟行为）？
    - 我是否在需要时遵循了 TDD？
    - 测试是否全面？

    如果在自我审查中发现问题，报告之前修复它们。

    ## 报告格式

    完成后报告：
    - 你实现了什么
    - 你测试了什么及测试结果
    - 修改的文件
    - 自我审查发现（如果有）
    - 任何问题或疑虑
```
