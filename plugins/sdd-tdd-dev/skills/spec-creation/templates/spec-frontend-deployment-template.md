## 2. CDN 和缓存策略

### 2.1 CDN 配置
| 资源类型 | CDN域名 | 缓存策略 | 备注 |
|---------|---------|---------|------|
| JS/CSS | cdn.example.com | max-age=31536000, immutable | 带hash文件名 |
| HTML | cdn.example.com | no-cache | 入口文件 |
| 静态资源 | cdn.example.com | max-age=31536000 | 图片、字体等 |
| API | api.example.com | - | 接口域名 |

### 2.2 HTTP 缓存策略
| 资源类型 | Cache-Control | ETag | 原因 |
|---------|--------------|------|------|
| HTML入口 | `no-cache, no-store, must-revalidate` | ✓ | 确保最新版本 |
| JS/CSS Bundle | `public, max-age=31536000, immutable` | - | 文件名带hash，永久缓存 |
| 图片 | `public, max-age=31536000` | ✓ | 不常变资源 |
| 字体 | `public, max-age=31536000, immutable` | - | 极少变更 |

## 4. 回滚方案

### 4.1 回滚触发条件
- [ ] 页面白屏或无法访问
- [ ] 核心功能异常，影响用户使用
- [ ] JS错误率超过 [X]%
- [ ] 页面加载时间明显变慢
- [ ] 大量用户投诉

### 4.2 回滚步骤
| 步骤 | 操作 | 预计耗时 |
|------|------|---------|
| 1 | 切换至上一版本构建产物 | 1min |
| 2 | 回滚HTML入口文件 | 30s |
| 3 | 清除CDN缓存 | 2min |
| 4 | 验证回滚成功 | 5min |

### 4.3 回滚注意事项
- 前端回滚通常只需要替换HTML入口文件和静态资源
- HTML文件名不含hash，需要强制刷新CDN
- JS/CSS文件名带hash，旧版本仍然有效，无需处理
