# 前端监控和可观测性方案

## 1. 错误追踪

### 1.1 JS 运行时错误
| 错误类型 | 捕获方式 | 上报内容 | 告警策略 |
|---------|---------|---------|---------|
| 语法错误 | `window.onerror` | 文件名、行号、堆栈 | 新增错误 > 10次/h |
| Promise拒绝 | `unhandledrejection` | Promise reject值 | 同上 |
| 异步错误 | `try-catch` 包裹 | 错误信息、上下文 | 同上 |
| 资源加载失败 | `onerror` 事件 | 资源URL、类型 | 同上 |

### 1.2 接口异常
| 异常类型 | 捕获方式 | 上报内容 | 告警策略 |
|---------|---------|---------|---------|
| HTTP错误 | Axios拦截器/原生fetch包装 | URL、状态码、请求参数、响应体 | 错误率 > 5% (5min) |
| 超时 | 超时拦截器 | URL、超时阈值 | 超时率 > 2% |
| 网络异常 | fetch error | URL、错误类型 | 同上 |

### 1.3 错误上下文信息
错误上报必须包含：
- 时间戳
- 用户ID（已登录时）
- Session ID
- 浏览器信息（User Agent）
- 操作系统
- 页面URL
- 设备类型（移动端/桌面端）
- 网络状态（4G/5G/WiFi）
- 错误堆栈
- 关键业务上下文

## 2. 性能指标

### 2.1 Core Web Vitals
| 指标 | 含义 | 目标值 | 采集方式 |
|------|------|--------|---------|
| LCP | 最大内容绘制 | < 2.5s | `PerformanceObserver` |
| FID/INP | 首次输入延迟/交互到下一帧 | FID < 100ms / INP < 200ms | `PerformanceObserver` |
| CLS | 累积布局偏移 | < 0.1 | `PerformanceObserver` |

### 2.2 其他关键指标
| 指标 | 含义 | 目标值 | 采集方式 |
|------|------|--------|---------|
| FCP | 首次内容绘制 | < 1.8s | `performance.getEntriesByType('paint')` |
| TTFB | 首字节时间 | < 800ms | `performance.timing` |
| 页面完全加载时间 | 页面完全可交互 | < 5s | `window.onload` |
| 首屏渲染时间 | 首屏内容可见 | < 3s | 自定义埋点 |

### 2.3 资源加载性能
| 资源类型 | 监控内容 | 告警阈值 |
|---------|---------|---------|
| JS Bundle | 文件大小、加载时间 | 单个Bundle > 500KB |
| CSS | 文件大小、加载时间 | 单个CSS > 100KB |
| 图片 | 文件大小、懒加载效率 | 单张图片 > 2MB |
| API请求 | 响应时间、数据量 | P95 > 2s |

|------|---------|---------|
| PV | 页面加载完成 | 页面URL、页面标题、Referrer |
| UV | Session内首次访问 | 用户ID、设备ID |
| 停留时长 | 页面卸载 | 页面URL、停留时间 |

### 3.2 交互行为
| 埋点 | 触发时机 | 上报内容 |

| 页面跳转 | route change | 来源页面、目标页面 |
| 搜索 | 搜索提交 | 搜索关键词、结果数量、耗时 |

## 5. 告警规则

### 5.1 告警等级
| 等级 | 条件 | 通知方式 | 响应时间 |
|------|------|---------|---------|
| P0 | JS错误率 > 10% (5min窗口) | 钉钉+短信+电话 | 立即 |
| P1 | JS错误率 > 5% (5min窗口) | 钉钉+短信 | 15min |
| P2 | P95页面加载 > 5s | 钉钉 | 30min |
| P3 | Core Web Vitals指标不达标 | 钉钉日报 | 当日 |

### 5.2 告警聚合
- 错误按类型聚合，避免告警风暴
- 同一错误5分钟内只告警一次
- 告警恢复通知：错误率恢复正常时
