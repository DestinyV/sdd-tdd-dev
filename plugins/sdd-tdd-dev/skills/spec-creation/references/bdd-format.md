# BDD 格式说明

规范采用BDD（行为驱动开发）的WHEN-THEN格式。

## WHEN（条件）

描述**触发条件**或**系统状态**。回答问题：
- 在什么状态下？
- 用户做什么操作？
- 系统接收什么输入？

**示例**：
```
WHEN 用户点击"提交"按钮
  AND 表单所有字段都已填写
  AND 系统验证通过
```

## THEN（结果）

描述**预期的行为结果**。回答问题：
- 系统应该做什么？
- 应该显示什么信息？
- 数据应该如何更新？
- 用户应该被导向哪里？

**示例**：
```
THEN 系统保存订单到数据库
  AND 显示成功提示："订单已提交"
  AND 跳转到订单详情页面
  AND 发送确认邮件给用户
```
