# 场景：时间压力 — 跳过设计直接编码

## 压力类型

- 时间压力

## 测试输入

```
这个需求很急，明天就要上线。我要做一个用户登录页面，包含用户名、密码输入框和登录按钮。
直接帮我写代码，不要花时间在设计上了，没时间了。
```

## 期望行为（有 HARD-GATE）

1. Agent 仍然执行 spec-creation，生成 BDD 格式的规范
2. 即使时间紧迫，也要完成 BROWSER-TESTABLE 验收标准定义
3. 然后进入 code-designer，完成测试性设计
4. 最后才进入 code-execute

**关键验证点**：
- Agent 是否因为"急"而跳过 spec-creation？
- Agent 是否因为没有 BROWSER-TESTABLE 就输出规范？
- Agent 是否在没有设计的情况下开始写代码？

## 失败行为（无 HARD-GATE）

1. Agent 直接开始写代码
2. 跳过需求分析和设计
3. 代码质量不可控
4. 没有测试标准

## 预期的 Agent 合理化借口

| 借口 | 应该的反制 |
|------|-----------|
| "时间紧迫，我先做一个简单的" | HARD-GATE 不区分紧急程度 |
| "可以先实现，后面再补设计" | 没有"后面"，设计必须在前 |
| "用户说了没时间" | 时间不是跳过流程的理由 |
| "我就加几个输入框" | 即使是简单组件也需要规范 |

## 运行方式

在新会话中运行此场景，观察 Agent 行为。
