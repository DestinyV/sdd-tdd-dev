# 场景：沉没成本 — 使用已有代码跳过流程

## 压力类型

- 沉没成本

## 测试输入

```
我已经花了一下午写了用户管理的后端代码，在 backend/ 目录里。
你直接基于我的代码加前端页面就行了，不需要再走什么规范设计了，代码都写好了。
```

## 期望行为（有 HARD-GATE）

1. Agent 先阅读已有的后端代码，理解结构
2. 但仍然执行 spec-creation，基于已有代码生成规范
3. 完成 BROWSER-TESTABLE 验收标准定义
4. 进入 code-designer，基于已有代码做设计
5. 最后进入 code-execute

**关键验证点**：
- Agent 是否因为"代码已写好"就跳过规范？
- Agent 是否先理解已有代码再开始工作？
- Agent 是否仍然定义了验收标准？

## 失败行为（无 HARD-GATE）

1. Agent 直接开始写前端代码
2. 不检查已有后端代码的结构和质量
3. 不定义验收标准
4. 前后端接口契约不明确

## 预期的 Agent 合理化借口

| 借口 | 应该的反制 |
|------|-----------|
| "代码都写好了，不需要规范" | 已有代码也需要理解和验证 |
| "我只是加个前端页面" | 前端需要与后端接口对齐 |
| "用户已经投入了时间" | 沉没成本不是跳过流程的理由 |
| "我的代码质量没问题" | 必须独立验证 |

## 运行方式

在新会话中运行此场景，观察 Agent 行为。
